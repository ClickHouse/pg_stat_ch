# pg_stat_ch – Single-pipeline Postgres 16+ Query Insights Exporter (to ClickHouse)

**Goal:** Build a PostgreSQL 16+ extension (`pg_stat_ch`) that captures query execution telemetry via server hooks and exports it to ClickHouse using **one** data pipeline:
`hooks (foreground) → shared-memory queue → bgworker exporter → ClickHouse events_raw`.
All dashboards/rollups/notables are derived in ClickHouse (materialized views / queries).

This spec is written for an implementation in **C++** (using the Postgres extension C ABI).

---

## 1. Requirements

### 1.1 Functional
- Capture execution telemetry for:
  - Executor statements: `SELECT`, `INSERT`, `UPDATE`, `DELETE`.
  - Utility statements: DDL, `VACUUM`, `ANALYZE`, `SET`, etc.
- Record both successful and failed statements with error metadata.
- Export telemetry to ClickHouse in batches, without impacting query latency.

### 1.2 Non-functional
- **Performance:** No network I/O on the query path. Small constant overhead per captured statement.
- **Correctness:** Robust error capture (SQLSTATE + message) for executor and utility.
- **Bounded memory:** Shared-memory queue is bounded; overflow results in dropped events with counters.
- **Operational:** Configurable sampling/thresholds. Introspection counters.

### 1.3 Compatibility
- PostgreSQL **16+** (major versions >= 16).
- Linux/macOS build supported by PGXS toolchain.
- ClickHouse reachable via native protocol (9000) using clickhouse-cpp SDK.

### 1.4 Dependencies
- **clickhouse-cpp** - C++ client library for ClickHouse (native protocol)
- Dependencies are vendored as git submodules in `third_party/`

---

## 2. Architecture (Single Pipeline)

### 2.1 Dataflow
1. **Foreground hooks**
   - Capture start snapshot and end deltas.
   - Use backend-local state to accumulate stats before committing to shared memory.
   - Create an `Event` struct and enqueue into shmem ring at query end.
2. **Shared-memory ring buffer**
   - Multi-producer (backends), single consumer (exporter BGWorker).
3. **Background worker**
   - Pops events in batches.
   - Resolves `dbid/userid` to names (via syscache lookups).
   - Inserts batch to ClickHouse using clickhouse-cpp native protocol.
4. **ClickHouse**
   - Stores raw events in `pg_stat_ch.events_raw`.
   - Optional materialized views compute aggregates and notables.

### 2.2 Why single-pipeline
- The extension emits **only** raw events.
- Aggregation (p50/p95/p99/top queries/errors) is done in ClickHouse.

## 3. Postgres Instrumentation Plan (Detailed)

### 3.1 Complete Hooks to Implement

Based on analysis of pg_stat_monitor, we implement the following hooks:

| Hook | Purpose | Performance Impact |
|------|---------|-------------------|
| `shmem_request_hook` (PG15+) | Request shared memory at startup | None (startup only) |
| `shmem_startup_hook` | Initialize shared memory structures | None (startup only) |
| `post_parse_analyze_hook` | Capture queryId early, store normalized query | Minimal |
| `planner_hook` | (Optional) Capture planning time & buffer usage | ~1-5μs |
| `ExecutorStart_hook` | Snapshot rusage, buffer/WAL state; allocate instrumentation | ~1μs |
| `ExecutorRun_hook` | Track nesting level, wrap in PG_TRY/PG_CATCH | ~0.5μs |
| `ExecutorFinish_hook` | Track nesting for proper cleanup | ~0.5μs |
| `ExecutorEnd_hook` | Finalize stats, compute deltas, enqueue event | ~2-5μs |
| `ProcessUtility_hook` | Handle DDL/utility statements with timing | ~2-5μs |
| `emit_log_hook` | Capture errors asynchronously (WARNING+) | ~1μs on error path |
| `ExecutorCheckPerms_hook` | (Optional) Capture relations touched | ~1μs |

### 3.2 Hook Implementation Details

#### 3.2.1 `post_parse_analyze_hook` – Early Query Capture
```c
// Called after parsing, before planning
// Signature (PG14+): void hook(ParseState *pstate, Query *query, JumbleState *jstate)

- Capture query->queryId (generated by PG14+ if compute_query_id=on)
- If queryId == 0, assign 1 to avoid confusion with utility statements
- Generate normalized query using JumbleState if normalized_query enabled
- Create local entry and add to backend-local list (NOT shmem yet)
- Store cmd_type from query->commandType
```

Key insight: Create entry early but don't touch shared memory until execution completes.

#### 3.2.2 `ExecutorStart_hook` – Initialize Instrumentation
```c
// Signature: void hook(QueryDesc *queryDesc, int eflags)

- Call getrusage(RUSAGE_SELF, &rusage_start) for CPU time tracking
- Call prev hook or standard_ExecutorStart()
- If query should be tracked (enabled && queryId != 0):
  - Allocate Instrumentation in per-query context:
    queryDesc->totaltime = InstrAlloc(1, INSTRUMENT_ALL, false);
  - This enables automatic buffer/WAL tracking
```

Key insight: Use `InstrAlloc()` to leverage PostgreSQL's built-in instrumentation.

#### 3.2.3 `ExecutorRun_hook` – Nesting & Error Handling
```c
// Signature (PG18+): void hook(QueryDesc *queryDesc, ScanDirection direction, uint64 count)
// Signature (<PG18): void hook(QueryDesc *queryDesc, ScanDirection direction, uint64 count, bool execute_once)

- Track nested query IDs for parent-child correlation:
  if (nesting_level < max_stack_depth) {
    nested_queryids[nesting_level] = queryDesc->plannedstmt->queryId;
    nested_query_txts[nesting_level] = strdup(queryDesc->sourceText);
  }
- nesting_level++;
- PG_TRY() {
    prev_hook or standard_ExecutorRun();
    nesting_level--;
    // cleanup nested_queryids on success
  }
  PG_CATCH() {
    nesting_level--;
    // cleanup nested_queryids
    PG_RE_THROW();  // re-throw, emit_log_hook will capture error
  }
  PG_END_TRY();
```

Key insight: Use `PG_TRY/PG_CATCH` only for nesting cleanup; let `emit_log_hook` capture errors.

#### 3.2.4 `ExecutorEnd_hook` – Finalize & Store
```c
// Signature: void hook(QueryDesc *queryDesc)

if (queryId != 0 && queryDesc->totaltime && enabled) {
  // Finalize instrumentation
  InstrEndLoop(queryDesc->totaltime);

  // Get CPU time delta
  getrusage(RUSAGE_SELF, &rusage_end);
  sys_info.utime = time_diff(rusage_end.ru_utime, rusage_start.ru_utime);
  sys_info.stime = time_diff(rusage_end.ru_stime, rusage_start.ru_stime);

  // All buffer/WAL/timing data available from queryDesc->totaltime:
  // - totaltime->total (execution time in seconds)
  // - totaltime->bufusage (BufferUsage struct)
  // - totaltime->walusage (WalUsage struct)

  // JIT info from estate (PG15+):
  // - queryDesc->estate->es_jit->instr

  // Parallel worker counts (PG18+):
  // - queryDesc->estate->es_parallel_workers_to_launch
  // - queryDesc->estate->es_parallel_workers_launched

  // Row count:
  // - queryDesc->estate->es_processed

  // Store entry (enqueue to shmem ring)
  enqueue_event(entry);
}

prev_hook or standard_ExecutorEnd();
```

#### 3.2.5 `ProcessUtility_hook` – Utility Statements
```c
// Signature (PG14+): void hook(PlannedStmt *pstmt, const char *queryString, bool readOnlyTree,
//                              ProcessUtilityContext context, ParamListInfo params,
//                              QueryEnvironment *queryEnv, DestReceiver *dest, QueryCompletion *qc)

// Skip EXECUTE/PREPARE/DEALLOCATE to avoid double-counting
if (IsA(parsetree, ExecuteStmt) || IsA(parsetree, PrepareStmt) || IsA(parsetree, DeallocateStmt))
  return call_prev_or_standard();

// Snapshot buffer/WAL/time at start
BufferUsage bufusage_start = pgBufferUsage;
WalUsage walusage_start = pgWalUsage;
INSTR_TIME_SET_CURRENT(start);
getrusage(RUSAGE_SELF, &rusage_start);

nesting_level++;
PG_TRY() {
  prev_hook or standard_ProcessUtility();
  nesting_level--;
}
PG_CATCH() {
  nesting_level--;
  PG_RE_THROW();  // emit_log_hook captures error
}
PG_END_TRY();

// Compute deltas
INSTR_TIME_SET_CURRENT(duration);
INSTR_TIME_SUBTRACT(duration, start);
BufferUsageAccumDiff(&bufusage, &pgBufferUsage, &bufusage_start);
WalUsageAccumDiff(&walusage, &pgWalUsage, &walusage_start);

// Row count for COPY/FETCH/SELECT/REFRESH
rows = (qc && (qc->commandTag == CMDTAG_COPY || ...)) ? qc->nprocessed : 0;

// Store entry
enqueue_event(entry);
```

#### 3.2.6 `emit_log_hook` – Asynchronous Error Capture
```c
// Signature: void hook(ErrorData *edata)

if (edata == NULL || IsParallelWorker() || MyProc == NULL)
  goto call_prev;

// Only capture WARNING and above
if (edata->elevel >= WARNING && !disable_error_capture) {
  // Get query text from debug_query_string (global)
  const char *query = debug_query_string ? debug_query_string : "";

  // Store error with SQLSTATE
  entry->err_sqlstate = unpack_sql_state(edata->sqlerrcode);  // e.g., "42P01"
  entry->err_msg = edata->message;
  entry->err_elevel = edata->elevel;

  enqueue_error_event(query, edata);
}

// Re-enable error capture after ERROR
if (edata->elevel >= ERROR)
  disable_error_capture = false;

call_prev:
if (prev_emit_log_hook)
  prev_emit_log_hook(edata);
```

Key insight: Use `disable_error_capture` flag to prevent deadlocks when holding LWLock.

#### 3.2.7 `planner_hook` – (Optional) Planning Time
```c
// Signature (PG14+): PlannedStmt *hook(Query *parse, const char *query_string,
//                                      int cursorOptions, ParamListInfo boundParams)

if (track_planning && query_string && queryId != 0) {
  BufferUsage bufusage_start = pgBufferUsage;
  WalUsage walusage_start = pgWalUsage;
  INSTR_TIME_SET_CURRENT(start);

  nesting_level++;
  PG_TRY() {
    result = prev_hook or standard_planner();
  }
  PG_FINALLY() {
    nesting_level--;
  }
  PG_END_TRY();

  // Compute planning time delta
  INSTR_TIME_SET_CURRENT(duration);
  INSTR_TIME_SUBTRACT(duration, start);

  // Store planning stats (can be merged into event later)
  local_entry->plan_time_us = INSTR_TIME_GET_MICROSEC(duration);
  local_entry->plan_bufusage = delta(pgBufferUsage, bufusage_start);
}
```

## 4. Capture Policy (Volume Control)

To keep the single pipeline performant, use **capture gating**:

- `capture_rate` (0..1): probabilistic sampling per statement.
- `capture_min_duration_us`: capture only if duration >= threshold (except errors).
- `top_level_only`: default true.
- `capture_query`: `off | sampled | all | errors_only`
  - `sampled`: include query text only for events that pass sampling and/or are "notable"
  - "notable" = error OR duration >= `slow_us` OR rows >= `rows_threshold`

### 4.1 Recommended defaults (prod-safe)
- `capture_rate = 1.0` (capture all, let duration filter)
- `capture_min_duration_us = 1000` (1ms)
- `slow_us = 100000` (100ms)
- `rows_threshold = 10000`
- `capture_query = sampled`
- `top_level_only = true`
- `track_planning = false` (enable for debugging)
- `track_utility = true`

### 4.2 Sampling Decision Flow
```c
bool should_capture(uint64 duration_us, uint64 rows, bool is_error) {
  if (IsParallelWorker()) return false;
  if (track_level == TRACK_NONE) return false;
  if (track_level == TRACK_TOP && nesting_level > 0) return false;

  // Always capture errors
  if (is_error) return true;

  // Apply sampling rate
  if (pg_prng_double(&prng_state) > capture_rate) return false;

  // Apply duration threshold
  if (duration_us < capture_min_duration_us) return false;

  return true;
}

bool is_notable(uint64 duration_us, uint64 rows, bool is_error) {
  return is_error || duration_us >= slow_us || rows >= rows_threshold;
}
```

## 5. Event Schema (Extension → ClickHouse)

### 5.1 Event struct (in shmem queue)
Keep it compact; truncate strings. Target size: ~512 bytes for fixed fields + query text.

```c
typedef struct Event {
  // Timing
  TimestampTz ts_start;           // 8 bytes
  uint64      duration_us;        // 8 bytes
  uint64      plan_duration_us;   // 8 bytes (optional, 0 if not tracking)

  // Identity
  Oid         dbid;               // 4 bytes
  Oid         userid;             // 4 bytes
  uint32      pid;                // 4 bytes
  uint64      queryid;            // 8 bytes
  uint8       top_level;          // 1 byte
  uint8       cmd_type;           // 1 byte (CmdType enum)
  uint16      _padding;           // 2 bytes

  // Rows
  uint64      rows;               // 8 bytes

  // Error info
  char        err_sqlstate[6];    // 5 chars + null (e.g., "42P01")
  uint8       err_elevel;         // 1 byte (0=success, WARNING=19, ERROR=20, FATAL=21)
  uint8       _padding2;          // 1 byte

  // Buffer usage
  int64       shared_blks_hit;
  int64       shared_blks_read;
  int64       shared_blks_dirtied;
  int64       shared_blks_written;
  int64       local_blks_hit;
  int64       local_blks_read;
  int64       local_blks_dirtied;
  int64       local_blks_written;
  int64       temp_blks_read;
  int64       temp_blks_written;

  // I/O timing (microseconds)
  int64       shared_blk_read_time_us;
  int64       shared_blk_write_time_us;
  int64       temp_blk_read_time_us;
  int64       temp_blk_write_time_us;

  // WAL usage
  int64       wal_records;
  int64       wal_fpi;
  uint64      wal_bytes;

  // CPU time (microseconds)
  int64       cpu_user_time_us;
  int64       cpu_sys_time_us;

  // JIT (optional, set to 0 if not available)
  int32       jit_functions;
  int32       jit_generation_time_us;
  int32       jit_inlining_time_us;
  int32       jit_optimization_time_us;
  int32       jit_emission_time_us;

  // Parallel workers (PG18+)
  int16       parallel_workers_planned;
  int16       parallel_workers_launched;

  // Variable-length fields (stored separately or inline with length prefix)
  uint16      appname_len;        // length of appname
  uint16      client_addr_len;    // length of client_addr
  uint16      query_len;          // length of query text
  uint16      err_msg_len;        // length of error message

  // Followed by: appname, client_addr, query, err_msg (if any)
  // char data[];  // flexible array member
} Event;
```

## 6. ClickHouse-derived Views (No extra extension pipeline)

### 6.1 Aggregates (materialized view)
Choose approximate quantiles for scale (TDigest), or Exact for smaller volumes.

```sql
CREATE MATERIALIZED VIEW pg_stat_ch.agg_1m
ENGINE = AggregatingMergeTree
PARTITION BY toDate(bucket_start)
ORDER BY (bucket_start, db, "user", query_id, cmd_type, app)
AS
SELECT
  toStartOfMinute(ts_start) AS bucket_start,
  db, "user", query_id, cmd_type, app,
  countState() AS calls,
  sumState(duration_us) AS total_time_us,
  minState(duration_us) AS min_time_us,
  maxState(duration_us) AS max_time_us,
  quantileTDigestState(0.50)(duration_us) AS p50_us,
  quantileTDigestState(0.95)(duration_us) AS p95_us,
  quantileTDigestState(0.99)(duration_us) AS p99_us,
  sumState(rows) AS total_rows,
  sumState(err_sqlstate != '') AS errors,
  sumState(shared_blks_hit) AS total_shared_blks_hit,
  sumState(shared_blks_read) AS total_shared_blks_read,
  sumState(wal_bytes) AS total_wal_bytes
FROM pg_stat_ch.events_raw
GROUP BY bucket_start, db, "user", query_id, cmd_type, app;
```

Read with `...Merge()` functions.

### 6.2 Notables (view)
```sql
CREATE VIEW pg_stat_ch.notable AS
SELECT *
FROM pg_stat_ch.events_raw
WHERE err_sqlstate != ''
   OR duration_us >= 100000      -- 100ms
   OR rows >= 10000;
```

### 6.3 Errors summary
```sql
CREATE VIEW pg_stat_ch.errors AS
SELECT
  toStartOfHour(ts_start) AS hour,
  db,
  err_sqlstate,
  err_elevel,
  count() AS error_count,
  any(err_msg) AS sample_message,
  any(query) AS sample_query
FROM pg_stat_ch.events_raw
WHERE err_sqlstate != ''
GROUP BY hour, db, err_sqlstate, err_elevel
ORDER BY hour DESC, error_count DESC;
```

## 7. Shared-memory Queue

### 7.1 Ring buffer design
MPSC (Multi-Producer, Single-Consumer) with fine-grained locking.

```c
typedef struct RingBuffer {
  LWLock     *lock;              // Protects head/tail modifications
  pg_atomic_uint64 head;         // Write position (atomic for readers)
  pg_atomic_uint64 tail;         // Read position
  uint64     capacity;           // Number of slots
  pg_atomic_uint64 dropped;      // Counter for dropped events
  pg_atomic_uint64 enqueued;     // Counter for successfully enqueued events
  // Event slots follow
  // char data[];  // Event data area
} RingBuffer;
```

### 7.2 Lock-free optimization (v2)
For high-throughput scenarios, consider lock-free MPSC queue:
- Use compare-and-swap for head advancement
- Single consumer doesn't need lock for tail

### 7.3 Sizing
- `queue_capacity_events` default: 65536
- Event size: ~512 bytes fixed + variable (query ~2KB max)
- Total queue size: ~170MB with default settings
- Consider: separate fixed and variable-length storage

### 7.4 Overflow handling
```c
bool enqueue_event(Event *event) {
  uint64 head = pg_atomic_read_u64(&ring->head);
  uint64 tail = pg_atomic_read_u64(&ring->tail);

  if (head - tail >= ring->capacity) {
    // Queue full - drop event and increment counter
    pg_atomic_fetch_add_u64(&ring->dropped, 1);
    return false;
  }

  LWLockAcquire(ring->lock, LW_EXCLUSIVE);
  // ... copy event to slot ...
  pg_atomic_fetch_add_u64(&ring->head, 1);
  pg_atomic_fetch_add_u64(&ring->enqueued, 1);
  LWLockRelease(ring->lock);
  return true;
}
```

## 8. Background Worker Exporter

### 8.1 Responsibilities
- Wake every `flush_interval_ms` (default: 1000).
- Pop up to `batch_max` events (default: 1000).
- Convert:
  - `dbid` → db name (via `get_database_name()`)
  - `userid` → role name (via `GetUserNameFromId()`)
  - `cmd_type` enum → string ("SELECT", "INSERT", etc.)
- Insert batch to ClickHouse using clickhouse-cpp native protocol.

### 8.2 clickhouse-cpp Integration
```cpp
#include <clickhouse/client.h>

// Connection managed by bgworker, reconnects on failure
clickhouse::Client client(clickhouse::ClientOptions()
    .SetHost(config.clickhouse_host)
    .SetPort(config.clickhouse_port)
    .SetUser(config.clickhouse_user)
    .SetPassword(config.clickhouse_password)
    .SetDefaultDatabase(config.clickhouse_database)
    .SetSendRetries(3)
    .SetRetryTimeout(std::chrono::seconds(5))
    .SetCompressionMethod(clickhouse::CompressionMethod::LZ4));

void insert_batch(Event *events, int count) {
  clickhouse::Block block;

  // Build columns
  auto ts_col = std::make_shared<clickhouse::ColumnDateTime64>(6);
  auto duration_col = std::make_shared<clickhouse::ColumnUInt64>();
  auto db_col = std::make_shared<clickhouse::ColumnString>();
  // ... other columns ...

  for (int i = 0; i < count; i++) {
    Event *e = &events[i];
    ts_col->Append(e->ts_start);
    duration_col->Append(e->duration_us);
    db_col->Append(get_database_name(e->dbid));
    // ... populate other columns ...
  }

  block.AppendColumn("ts_start", ts_col);
  block.AppendColumn("duration_us", duration_col);
  block.AppendColumn("db", db_col);
  // ... append other columns ...

  client.Insert("pg_stat_ch.events_raw", block);
}
```

### 8.3 Error Handling & Reliability
- Keep query path independent of exporter health.
- Exporter handles failures:
  - clickhouse-cpp provides built-in retry with `SetSendRetries()`
  - Additional retry with exponential backoff (1s, 2s, 4s, 8s, max 60s)
  - Do **not** block forever
- When ClickHouse is down:
  - Queue grows until full; then events drop
  - Counters indicate drops and last error
  - Log warning when drops start
- Connection management:
  - Reconnect on connection loss
  - Use LZ4 compression for efficient transfer

## 9. Configuration

### 9.1 GUC (PostgreSQL configuration)
Only runtime control is exposed as GUC:
```
pg_stat_ch.enabled = on          -- Master on/off switch (SIGHUP reloadable)
```

### 9.2 Config File (`pg_stat_ch.conf`)
All ClickHouse connection and operational parameters are read from a config file.
Location: `$PGDATA/pg_stat_ch.conf` (or specified via `pg_stat_ch.config_file` GUC).

```ini
# ClickHouse connection
clickhouse_host = localhost
clickhouse_port = 9000
clickhouse_user = default
clickhouse_password =
clickhouse_database = pg_stat_ch

# Queue settings
queue_capacity_events = 65536
max_query_len = 2048
max_err_msg_len = 256

# Export settings
flush_interval_ms = 1000
batch_max = 1000

# Capture policy
capture_rate = 1.0
capture_min_duration_us = 1000
slow_us = 100000
rows_threshold = 10000
capture_query = sampled          # off|sampled|all|errors_only
top_level_only = true
track_utility = true
track_planning = false
track = top                      # none|top|all
```

### 9.3 Config File Parsing
- Config file is read at bgworker startup and on SIGHUP.
- Simple INI-style format (key = value).
- Comments start with `#`.
- Missing file or keys use defaults.
- Invalid values logged as warnings, defaults used.

## 11. Security & PII Considerations

- Query text may contain secrets; default `capture_query=sampled` and keep strict truncation.
- Recommend application-level redaction for sensitive workloads.
- Provide `capture_query=off` to run in strict environments.
- Error messages can also contain values; truncate and allow disabling.
- Consider: regex-based redaction patterns (future enhancement).

## 12. Build & Deploy Notes

### 12.1 Dependencies (third_party/)
Dependencies are vendored as git submodules in `third_party/`:
```bash
# Add clickhouse-cpp as submodule
git submodule add https://github.com/ClickHouse/clickhouse-cpp.git third_party/clickhouse-cpp

# Initialize submodules after clone
git submodule update --init --recursive
```

Directory structure:
```
third_party/
└── clickhouse-cpp/     # ClickHouse C++ client library
```

### 12.2 Build
- Use CMake with PGXS integration.
- Export C ABI symbols via `extern "C"` in C++.
- Avoid STL in shared memory; use Postgres allocators and shmem APIs.
- Use `#if PG_VERSION_NUM >= XXXXX` for version-specific code.
- clickhouse-cpp is built as part of the extension build:
  ```cmake
  add_subdirectory(third_party/clickhouse-cpp)
  target_link_libraries(pg_stat_ch PRIVATE clickhouse-cpp-lib)
  ```

### 12.3 Deploy
1. `shared_preload_libraries = 'pg_stat_ch'`
2. Create config file `$PGDATA/pg_stat_ch.conf` with ClickHouse connection details.
3. Restart Postgres.
4. `CREATE EXTENSION pg_stat_ch;`
5. Create ClickHouse tables and views.

### 12.4 Version compatibility macros
